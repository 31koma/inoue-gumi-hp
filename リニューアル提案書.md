# 株式会社井上組 ホームページ リニューアル提案書

京都・右京区の鳶・足場工事「株式会社井上組」公式サイトを、既存の Next.js サイトをベースに全面ブラッシュアップしました。本書は要望いただいた12項目（①改善点〜⑫公開前チェックリスト）をまとめたものです。実装済みコードはこのフォルダ（`app/` `components/` `lib/`）に反映されています。

---

## ① 現在のホームページの改善点一覧

実際のソースコードを分析した結果、以下を改善対象としました。

**伝わらない／情報不足**
- トップの巨大コピー「組む。」＋英語「Invisible work. Visible trust.」だけでは、**何の会社か初見で分からない**（足場会社だと伝わらない）。→ 廃止。
- 会社情報が「株式会社 井上組（仮）」「住所を後ほど追記」など**プレースホルダーのまま**。→ 名刺の実情報（所在地・許可番号・代表者・連絡先）を反映。
- サービスが「単管足場／くさび式足場…」と**単語の羅列のみ**で、内容説明がない。→ 写真＋解説の事業内容ページへ。
- 施工実績・代表挨拶・採用・会社概要・お知らせの**ページが存在しない**（1ページのみ）。→ 8ページ構成へ拡張。

**ブランド・信頼性を損なう要素**
- **Berserk のアニメ曲「Gatts Baby」が自動再生**ボタンで常駐。建設会社の信頼感と著作権の両面でリスク。→ 完全に削除。
- 「WORKERS AS ICONS」「この現場に、来るか。」などコピーが**ファッション誌調**で、堅実さ・信頼感より奇抜さが先行。→ 落ち着いた高級トーンに刷新。
- 装飾用のCSSシルエット（人型）など、**意味のない演出**が多い。→ 写真主役の構成に置換。

**技術・UX・SEO**
- メタ情報が最小限（title/description のみ）、**構造化データ・OGP・sitemap・robots なし**。→ 全実装。
- **パンくず・固定CTA・問い合わせ動線・電話導線**がない。→ 全追加。
- 全要素が1ファイル（`page.tsx` 460行）に密結合で**保守しづらい**。→ データ層・共通部品に分離。
- 画像が `<img>` 直書きで**最適化されていない**。→ `next/image` で遅延読込・自動最適化。
- ほぼ黒背景に明るい黄 `#f7c600`（注意喚起色）で、**高級感より工事現場的**。→ 上品なゴールド/真鍮 `#c2a15b` に調整。

## ③ 新しいデザインコンセプト

**コンセプト：「静かな力強さ」— Quiet Strength**
Apple / Tesla / 海外建築会社のような「余白で語る高級感」と、鳶・足場会社の「現場の力強さ」を両立。

- **カラー**：黒（`#0b0c0e`）・チャコール（`#15171b`）・オフホワイト（`#f6f4ee`）をベースに、差し色は**上品なゴールド/真鍮（`#c2a15b`）**。名刺の金の印影に通じる、信頼と高級感の色。
- **フォント**：和文見出しに **Noto Serif JP**（明朝＝高級感・信頼）、数字・英字に **Oswald**（力強い無機質）、本文に **Noto Sans JP**（可読性）。
- **レイアウト**：大きな余白、グリッドの整列、写真を大きく使う。装飾は最小限。
- **動き**：スクロールでのフェードイン、数字のカウントアップ、画像のホバー拡大、ボタンのホバー。**派手すぎず**、`prefers-reduced-motion` にも配慮。

## ④ 新しいキャッチコピー候補（13案）と採用案

足場会社は「すべての工事の土台＝支える存在」という強い物語を持てます。これを軸に提案します。

1. **確かな足場で、未来を支える。** ★採用
2. すべての工事は、確かな足場から。
3. 街を支える、足場のプロフェッショナル。
4. 高さに、安心を架ける。
5. 現場の安全を、足元から支える。
6. 足場から、未来を組み立てる。
7. 信頼を組み、安全を支える。
8. すべての高さに、確かな一歩を。
9. 見えない技術が、街をつくる。
10. 安全という、最高の仕事を。
11. 足元から、街の未来を支える。
12. 高みへ挑む現場に、確かな足場を。
13. 京都の現場を、足元から。

**採用：「確かな足場で、未来を支える。」**
理由  — (1) 足場会社だと明確に伝わる、(2)「確かな（技術・品質）」「支える（信頼）」「未来（将来性・若さ）」を一文に凝縮、(3) 短くリズムが良く記憶に残る。サブコピーで事業・地域・資格を補足し、初見の理解を担保しています。

## ⑤ 各ページの構成

**TOP** … ①ヒーロー＋新コピー → ②井上組について（実績カウントアップ）→ ③事業内容 → ④施工実績 → ⑤代表メッセージ → ⑥採用 → ⑦会社概要 → ⑧お問い合わせCTA
**会社概要** … ヒーロー → 会社情報テーブル → 保有資格・許可 → アクセス＋Googleマップ → CTA
**代表挨拶** … ヒーロー → 代表写真（固定）＋本文（理念／創業／こだわり／品質／社員／お客様／ビジョン）→ CTA
**事業内容** … ヒーロー → 目次 → 6事業の写真付き詳細（左右交互レイアウト）→ CTA
**施工実績** … ヒーロー → カテゴリー絞込 → 写真主役カード（工事名・場所・内容・年月・コメント・複数枚）→ CTA
**採用情報** … ヒーロー → 求める人物像 → 働く魅力 → 一日の流れ → 福利厚生 → 先輩社員の声 → 募集要項 → 応募フォーム
**お知らせ** … ヒーロー → 記事一覧（日付・カテゴリ）→ 記事詳細
**お問い合わせ** … ヒーロー → 連絡先（電話/携帯/メール/受付/所在地）→ フォーム

## ⑦ UI・UX改善内容

- **スマホファースト**：全レイアウトをモバイル基準で設計、ブレークポイントで拡張。
- **固定ヘッダー**：スクロールで背景が引き締まる追従ヘッダー＋ハンバーガーメニュー。
- **固定CTA**：スマホは画面下に「採用／お問い合わせ」を2分割常駐、PCは右下に常駐。
- **電話導線**：ヘッダー・フッター・各所で `tel:` ワンタップ発信。
- **CTAを各ページ下部に配置**：共通 `CtaSection` を全ページ末尾に。
- **パンくずリスト**：全下層ページに設置（構造化データ付き）。
- **アニメーション**：スクロール・フェードイン、数字カウントアップ、画像ホバー拡大、ボタンホバー。
- **アクセシビリティ**：「本文へスキップ」リンク、`aria` 属性、フォーカスリング、`alt`、`prefers-reduced-motion` 対応。
- **表示速度**：`next/image` による最適化・遅延読込、フォントの `display: swap`、地図 iframe の遅延読込。

## ⑧ SEO改善内容

- **title 最適化**：`%s｜株式会社井上組` テンプレートで全ページ個別タイトル。
- **メタディスクリプション**：ページごとに地域・事業を含めて記述。
- **構造化データ（JSON-LD）**：`GeneralContractor`（会社情報）、`BreadcrumbList`（パンくず）、`NewsArticle`（記事）。
- **見出し構成**：各ページ `h1` は1つ、`h2`/`h3` を階層化。
- **内部リンク**：ヘッダー・フッター・本文・CTAで全ページを相互リンク。
- **画像 alt**：意味のある画像に説明、装飾画像は空 alt。
- **sitemap.xml / robots.txt**：自動生成（`app/sitemap.ts`・`app/robots.ts`）。
- **OGP / Twitter カード**：SNS共有時の見栄えを設定。
- **canonical**：各ページに正規URLを指定。
- **モバイル最適化**：レスポンシブ＋スマホファーストで評価を確保。

> ※ `lib/site.ts` の `url` を**公開ドメインに変更**すると、構造化データ・sitemap・OGP のURLが一括で正しくなります。

## ⑩ 必要な写真・素材一覧

現在は仮の装飾画像（`public/images/` の既存PNG）を流用しています。**プロ品質に仕上げるため、以下の実写真への差し替えを推奨**します（理想は横長・高解像度・自然光）。

| 用途 | 推奨内容 | 推奨サイズ |
|---|---|---|
| TOPヒーロー | 組み上がった足場の全景（できれば京都の現場） | 横2000px〜 |
| 事業内容（6点） | くさび式／単管／ビケ／鳶高所／施工計画／解体 の各作業写真 | 横1600px |
| 施工実績（各6件×複数枚） | 工事ごとの全景＋ディテール | 横1600px |
| 代表写真 | 代表・井上様のポートレート（現場 or スーツ） | 縦1200px |
| 先輩社員の声（2点〜） | 社員の人物写真 | 正方形1000px |
| 採用ヒーロー | 笑顔のチーム／作業風景 | 横2000px |
| ロゴ | 会社ロゴ（あれば。名刺の金の社章をベクター化推奨） | SVG/PNG透過 |
| favicon / OGP画像 | ロゴ＋社名（1200×630） | 各規定サイズ |

差し替えは `public/images/` に画像を置き、`lib/` 各データの `image`／`src` のパスを変えるだけです。

## ⑪ 実装コード

このフォルダに実装済みです。保守しやすい構成にしました。

```
app/                  各ページ（page.tsx）と SEO（sitemap.ts / robots.ts）
  layout.tsx          共通レイアウト・フォント・全体SEO・会社JSON-LD
  globals.css         デザインの土台（配色・アニメ・アクセシビリティ）
components/           再利用UI（ヘッダー/フッター/固定CTA/パンくず/フォーム 等）
lib/                  データの単一ソース（site/services/works/news）← 文章はここを編集
tailwind.config.ts   カラー・フォント定義
```

- **データとデザインを分離**：文章・実績・お知らせは `lib/` を編集するだけ。HTMLを触らずに更新可能。
- **共通コンポーネント化**：見出し・CTA・パンくず等を再利用し、トーンの統一と保守性を確保。
- 既存の Next.js / Tailwind / framer-motion 構成を**活かして**改修（新規ライブラリ追加なし）。

**ローカルでの確認**
```bash
npm run dev      # 開発サーバー（http://localhost:3000）
npm run build    # 本番ビルド
npm start        # 本番起動
```

**新しい実績・お知らせの追加例**：`lib/works.ts` または `lib/news.ts` の配列の先頭に1件追加するだけ。

### 補足：動作確認について
コードの型チェック（`tsc --noEmit`）はエラーなく通過しています。

`next build`（本番用ビルド）は、**お使いのMacでは問題なく使えます**。私の作業環境（クラウド側）だけは専用の実行ファイルをネットから取得できず、フルビルドを回せなかっただけです。公開前に一度だけ、お手元で `npm run build` を実行して最終確認していただくと安心です。

※ ふだんの表示確認は、`launch_inoue_gumi_site.command`（または `井上組サイト_テスト起動.command`）をダブルクリックするだけでOKです（開発サーバーが立ち上がり、ブラウザが自動で開きます）。
